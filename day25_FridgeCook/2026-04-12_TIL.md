# Day25 - FridgeCook Project

## 오늘 만든 것
오늘은 **냉장고 재료 기반 레시피 추천 웹 프로젝트**를 만들었다.  
사용자가 냉장고에 있는 재료를 직접 입력하거나 선택하면, 데이터베이스에 저장된 레시피와 비교해서 일치도가 높은 메뉴를 추천해주는 Flask 기반 웹 서비스다.

---

## 구현 내용

### 1. Flask 웹 애플리케이션 구성
- `Flask`를 사용해 메인 페이지 라우팅을 구성했다.
- `GET`, `POST` 요청을 구분해서 입력 페이지와 추천 결과를 처리했다.
- `render_template()`로 추천 결과와 재료 목록을 HTML 템플릿에 전달했다.

### 2. SQLite 데이터베이스 연동
- `sqlite3`를 이용해 레시피 데이터베이스를 연결했다.
- `row_factory = sqlite3.Row`를 사용해 컬럼명을 기준으로 데이터에 접근할 수 있도록 했다.
- 레시피 이름, 재료, 대표 이미지, 조리 단계 데이터를 불러와 Python에서 가공했다.

### 3. 재료 데이터 전처리
- 공공데이터 API의 재료 문자열은 그대로 쓰기 어려워서 정제 과정이 필요했다.
- 줄바꿈, 괄호, 단위, 불필요한 문자열을 제거했다.
- `replace()`와 `re.sub()`를 함께 사용해서 추천 로직에 활용 가능한 형태로 정리했다.

### 4. 재료 기반 추천 로직 구현
- 사용자가 가진 재료와 각 레시피 재료를 비교했다.
- 일치하는 재료가 있는 레시피만 추천 대상으로 삼았다.
- 일치한 재료 수를 전체 재료 수로 나누어 점수를 계산했다.
- 점수가 높은 순으로 정렬한 뒤 상위 12개 레시피를 보여주도록 했다.

### 5. 재료 카테고리 분류
- 전체 재료 목록을 카테고리별로 나누어 정리했다.
- 고기/계란, 수산물, 채소/버섯, 유제품/가공품, 양념/소스 등으로 분류했다.
- 사용자가 재료를 쉽게 찾을 수 있도록 사이드바 구조를 고려했다.

### 6. 조리 단계 확장 구조 설계
- 조리 단계는 `M1 ~ M20`, 이미지 컬럼은 `MI1 ~ MI20` 구조로 되어 있었다.
- 반복문을 사용해 최대 20단계까지 자동으로 불러오도록 구성했다.
- 단계 설명과 이미지 데이터를 함께 저장해 상세 조리 화면에 활용할 수 있게 했다.

---

## 오늘 배운 점
- Flask에서 사용자 입력을 받아 처리하는 기본 흐름을 익혔다.
- SQLite 데이터를 Python에서 가공해 웹 서비스에 연결하는 방식을 경험했다.
- 정규표현식으로 비정형 데이터를 정리하는 과정이 생각보다 중요하다는 걸 느꼈다.
- 단순 출력이 아니라, 입력 데이터를 기반으로 결과를 계산해 보여주는 로직을 직접 구현해봤다.
- 반복문을 활용해 확장 가능한 구조를 만드는 방법을 연습했다.

---

## 어려웠던 점
- 재료 문자열 형식이 일정하지 않아서 전처리 로직을 짜는 게 쉽지 않았다.
- 부분 문자열 매칭 방식이라 비슷한 재료끼리 잘못 매칭될 가능성이 있다.
- 추천 점수가 단순 비율 기반이라 실제 체감 정확도와 차이가 있을 수도 있다.

---

## 개선해보고 싶은 점
- 정규표현식을 더 정교하게 수정해서 전처리 정확도를 높이기
- 재료 동의어 처리나 정확 일치 로직 추가하기
- 추천 점수에 핵심 재료 가중치 반영하기
- UI를 더 다듬어서 실제 서비스처럼 개선하기
- 즐겨찾기, 최근 본 레시피 같은 기능 확장해보기

---

## 느낀 점
오늘 프로젝트는 단순한 문법 연습이 아니라,  
**데이터 전처리 + DB 연동 + 추천 로직 + 웹 화면 출력**을 하나로 연결해봤다는 점에서 의미가 컸다.

특히 사용자의 입력을 바탕으로 결과를 추천하는 흐름을 직접 구현해보면서  
이제 Flask 프로젝트도 조금씩 “서비스 형태”로 만들 수 있다는 자신감이 생겼다.
