## 1. Transformer 아키텍처 개요

Transformer는 기존의 CNN/RNN 기반 구조 대신  
**Attention 중심 구조**를 사용하는 모델이다.

자연어를 벡터로 변환하는 **Embedding** 과정을 거친 뒤,  
문맥 관계를 학습하는 여러 Attention 블록을 통과하며  
입력 문장의 의미를 점점 더 정교하게 표현한다.

또한 순서 정보가 사라지지 않도록 **Positional Encoding**을 추가해  
문장 내 단어 위치까지 함께 학습한다.

---

## 2. Transformer의 핵심 구성 요소

### Embedding
- 자연어 문장을 컴퓨터가 처리할 수 있도록 벡터로 변환하는 과정
- 입력 문장은 토큰 단위로 나뉘고, 각 토큰은 임베딩 벡터로 바뀐다

### Positional Encoding
- Transformer는 구조상 순서를 직접 기억하지 못하므로 위치 정보를 따로 더해줘야 한다
- 임베딩 벡터에 위치 인코딩 벡터를 더해 단어 순서를 반영한다

### Multi-Head Attention
- 여러 개의 Self-Attention을 병렬로 수행하는 방식
- 각 Head가 서로 다른 관점으로 문맥을 파악할 수 있어 표현력이 높아진다
- 하나의 Attention 연산 단위를 Head라고 부른다

### Position-wise Feed-Forward Network
- 각 위치의 표현을 더 정교하게 가공하는 신경망
- 보통 2층 퍼셉트론 구조와 ReLU 계열 활성함수를 사용한다

### Add & Norm
- 입력 벡터를 출력에 다시 더하는 **잔차 연결(Residual Connection)** 로 학습 안정성을 높인다
- **Layer Normalization**을 통해 각 레이어의 출력 스케일을 정규화한다

### Masked Multi-Head Attention
- 다음 단어 예측 시 미래 단어를 보지 못하도록 가리는 구조
- **Look-ahead Mask**를 사용해 치팅을 방지한다
- Attention Map의 상삼각 영역을 보지 못하도록 처리한다

### Cross Attention
- 디코더가 인코더의 출력을 참조할 수 있도록 하는 구조
- Query는 디코더 출력, Key/Value는 인코더 출력을 사용한다

### Verbalizer
- 디코더 마지막 출력에 선형 레이어와 소프트맥스를 적용해 최종 단어를 예측한다

---

## 3. 학습 데이터 전처리 방식

- 입력 데이터는 토큰들의 시계열 시퀀스로 구성된다
- 이전 단어들을 기반으로 다음 단어를 예측하도록 학습한다
- 문장 시작을 알리는 특수 토큰을 맨 앞에 넣어 첫 단어 예측이 가능하도록 만든다

---

## 4. 사전학습(Pre-training)

사전학습은 대규모 데이터로 Transformer를 먼저 학습시켜  
일반적인 언어 패턴을 익히게 하는 과정이다.

이후 특정 작업에 맞춰 추가 학습하는 **미세조정(Fine-tuning)** 을 통해  
실제 태스크에 적용할 수 있다.

### Masked Language Modeling (MLM)
- 일부 단어를 가리고 원래 단어를 맞히는 방식
- BERT의 대표적인 사전학습 기법
- 입력 토큰의 일부를 마스크, 랜덤 단어, 원래 단어로 변형해 학습한다

### Next Sentence Prediction (NSP)
- 두 문장이 실제로 이어지는 관계인지 예측하는 방식
- 문장 간 관계를 학습하는 데 사용된다

### Causal Language Modeling
- 왼쪽에서 오른쪽으로 순차적으로 다음 단어를 예측하는 방식
- GPT 계열의 대표적인 사전학습 방법이다

### 비교
- **MLM**: 마스크된 단어를 복원
- **NSP**: 두 문장의 연결 관계를 예측
- **Causal LM**: 이전 단어만 보고 다음 단어를 예측

---

## 5. 사전학습 기법 연구 사례

### RoBERTa
- BERT에서 NSP를 제거
- MLM만 사용
- 동적 마스킹 적용

### ELECTRA
- Generator가 일부 토큰을 바꾸고
- Discriminator가 해당 토큰이 원래 단어인지 판별
- 직접 다음 토큰을 맞히지 않고 진짜/가짜를 구분한다

### T5
- 모든 작업을 **텍스트 입력 → 텍스트 출력** 구조로 통일
- 인코더는 양방향, 디코더는 왼쪽에서 오른쪽 방식

### XLNet
- 순열 기반 학습 방식
- BERT의 양방향 장점과 GPT의 순차 예측 특성을 함께 반영

### ERNIE
- 개체명, 구문, 문장 관계 등 지식 정보를 함께 반영

### MPNet
- 양방향 문맥 이해와 순서 예측 개념을 결합한 방식

### GPT
- **Decoder Only** 구조
- 왼쪽에서 오른쪽으로 다음 토큰을 예측
- 챗봇, 생성형 AI, 에이전트 모델에 적합

---

## 6. Transformer의 미세조정(Fine-tuning)

미세조정은 사전학습된 모델을 특정 작업에 맞게 추가 학습하는 과정이다.

### 미세조정이 필요한 이유
- 사전학습 모델은 일반적인 언어 패턴만 학습한 상태다
- QA, 감정분석, 요약, 번역 등 실제 태스크에 맞게 조정이 필요하다
- 작업 성능을 더 높이기 위해 사용한다

### 기본 흐름
1. 모델 불러오기
2. 데이터 전처리
3. 아키텍처 튜닝
4. 미세조정 학습

---

## 7. 미세조정 전략

### Full Fine-tuning
- 전체 파라미터를 모두 업데이트
- 성능은 좋지만 리소스 소모가 큼

### Freeze Fine-tuning
- 일부 레이어만 학습하고 나머지는 고정
- 전체 미세조정보다 효율적

### Feature-based Tuning
- 사전학습 모델의 중간 출력 특징은 고정
- 분류기 등 후단 레이어만 학습

### Adapter Tuning
- 중간 계층에 작은 네트워크를 추가
- 원본 모델은 고정하고 추가 모듈만 학습

### Instruction Tuning
- 명령어를 더 잘 이해하도록 학습
- QA나 대화형 태스크에 효과적

### Reasoning Tuning
- 정답만이 아니라 사고 과정까지 반영하도록 학습
- 수학, 논리 문제 해결에 강점이 있다

## 8. 오늘의 실습 - 듀얼 인코더

오늘은 이론 학습과 함께 **듀얼 인코더(Dual Encoder)** 실습도 진행했다.

듀얼 인코더는 두 입력을 각각 인코더에 통과시켜  
각각의 벡터 표현으로 변환한 뒤,  
두 벡터의 유사도를 비교하는 방식이다.

### 특징
- 입력 두 개를 각각 독립적으로 임베딩한다
- 생성형 방식이 아니라 **비교/검색형 구조**에 가깝다
- 문장-문장, 질문-문서, 사용자-아이템 매칭 등에 활용할 수 있다

### 느낀 점
- 같은 Transformer 기반이라도 생성형 모델과 검색형 모델의 활용 방식이 다르다는 점이 인상적이었다
- 이론으로 배운 임베딩 개념이 실제 실습 구조와 연결되어 더 잘 이해됐다

## ✅ 회고

오늘은 Transformer를 단순히 “요즘 많이 쓰는 모델” 정도로 보는 것이 아니라,  
**내부 구조와 학습 방식, 그리고 실제 활용 방식까지 한 번에 연결해서 이해한 날**이었다.

특히 다음 내용이 중요하게 느껴졌다.

- Transformer는 Attention 중심 구조라는 점
- Positional Encoding으로 순서 정보를 보완한다는 점
- BERT 계열과 GPT 계열은 사전학습 방식이 다르다는 점
- 미세조정은 목적과 자원에 따라 전략이 달라진다는 점
- 듀얼 인코더는 생성이 아니라 임베딩 비교에 강한 구조라는 점
