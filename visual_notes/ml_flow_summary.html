머신러닝 흐름 요약
기초에서 실전까지

머신러닝은 모델 하나만 배우는 게 아니라, 데이터를 준비하고 문제를 정의하고 평가하는 전체 흐름을 이해하는 게 핵심입니다. 이 페이지는 데이터 → 전처리 → 학습 → 평가 → 개선 흐름을 압축해서 다시 보게 만드는 요약 챕터입니다.

핵심 단계 정리
Step 1

문제 정의

가장 먼저 해야 할 일은 “무엇을 예측할지”를 분명하게 정하는 것입니다.

  • 분류인지 회귀인지 먼저 구분
  • 입력(X)과 정답(y) 정의
  • 성공 기준이 되는 평가 지표 생각
스팸 분류
집값 예측
클러스터링
Step 2

데이터 수집과 확인

모델보다 먼저 데이터를 확인해야 합니다. 결측치, 이상치, 컬럼 의미 파악이 중요합니다.

  • shape, columns, dtypes 확인
  • 결측치와 이상치 체크
  • 타깃 분포와 데이터 편향 확인
Step 3

전처리

머신러닝 성능은 전처리 품질에 크게 좌우됩니다.

  • 결측치 처리
  • 범주형 인코딩
  • 스케일링 / 정규화
  • 학습/검증 데이터 분리
Step 4

특징 선택과 이해

모든 컬럼을 다 넣는 것보다, 의미 있는 특징을 찾는 게 중요합니다.

  • 상관관계 확인
  • 중요 변수 추정
  • 불필요한 컬럼 제거
  • 도메인 지식 반영
Step 5

모델 학습

문제 유형에 맞는 모델을 골라서 기본 성능을 먼저 확인합니다.

  • 분류: Logistic Regression, Decision Tree, RandomForest 등
  • 회귀: Linear Regression, Ridge, RandomForestRegressor 등
  • 비교할 baseline을 먼저 세우기
Step 6

평가와 개선

성능 숫자만 보는 게 아니라 왜 그렇게 나왔는지 해석해야 합니다.

  • 분류: accuracy, precision, recall, f1
  • 회귀: MAE, MSE, RMSE
  • 오버피팅 / 언더피팅 체크
  • 하이퍼파라미터 튜닝

Mindset

중요한 관점

머신러닝은 “모델 이름 외우기”보다 전체 흐름을 잡는 게 훨씬 중요합니다.

데이터를 보고
문제를 정의하고
전처리하고
평가하며 개선하기
한 줄 복습 포인트
핵심

데이터가 먼저다

모델보다 먼저 데이터 상태를 확인하고, 전처리와 특징 선택을 잘하는 게 성능에 큰 영향을 준다.

연결

딥러닝 전 챕터의 다리

머신러닝 흐름을 이해하면 CNN, RNN, Transformer 같은 뒤 챕터도 “학습 파이프라인” 관점에서 더 잘 보인다.
