# 2026-04-20 Speech AI (STT / TTS / Audio)

## 오늘 학습한 내용

### 1. 음성의 기초 개념
- 소리는 **진동으로 인한 공기의 압축**으로 전달된다.
- 음성을 이해할 때 중요한 요소는 다음과 같다.
  - **진폭(Amplitude)**: 소리의 세기
  - **주파수(Frequency)**: 떨림의 빠르기
  - **위상(Phase)**: 파동의 위치 정보

### 2. 물리 음향과 심리 음향
- **물리 음향**
  - 진폭의 세기
  - 떨림의 빠르기
  - 파동의 모양
- **심리 음향**
  - 소리 크기
  - 음정 / 높낮이
  - 음색
  - 사람이 느끼는 소리 감각

### 3. 주파수와 복합파
- 주파수의 단위는 **Hertz(Hz)** 를 사용한다.
- 주기는 파동이 한 번 진동하는 데 걸리는 시간이다.
- 우리가 실제로 사용하는 대부분의 소리는 단순한 정현파가 아니라,
  **여러 개의 서로 다른 정현파가 합쳐진 복합파**이다.

### 4. 푸리에 변환과 STFT
- **푸리에 변환(Fourier Transform)** 은 임의의 입력 신호를 다양한 주파수 성분으로 분해해서 표현하는 방법이다.
- 결과로는 실수부와 허수부를 가지는 복소수가 나오며,
  - 절대값은 주파수 강도
  - 위상은 주파수의 위치 정보를 의미한다.
- 단점은 **시간 정보가 사라진다**는 점이다.

- 이를 보완하기 위해 **STFT(Short-Time Fourier Transform)** 를 사용한다.
- STFT는 신호를 짧은 구간으로 나누어 분석하기 때문에
  **구간별 시간 정보와 주파수 정보를 함께 확인할 수 있다.**

## STT (Speech To Text)

### 5. 음성 데이터 이해
STT에서는 다음 요소를 함께 다룬다.

- **Waveform**: 음성 파일 자체
- **Utterance**: 발화 텍스트
- **Alignment**: 음성과 텍스트의 위치 정렬 정보

### 6. 음소와 스펙트로그램
- **Phoneme(음소)** 는 사람 목소리의 기본 발음 단위이다.
- 음성 처리는 보통 원시 waveform 그대로 쓰기보다
  **Spectrogram**, **Mel-Spectrogram** 형태로 변환해서 사용한다.
- STFT는 보통 짧은 구간 단위(예: 25ms / 10ms)로 분석한다.
- **Mel-frequency** 는 사람의 청각 특성을 반영해
  저주파는 촘촘하게, 고주파는 넓게 반영한다.

### 7. STT가 어려운 이유
- 지도학습을 적용하기 쉽지 않다.
- 음성과 텍스트의 **경계가 명확하지 않다.**
- 텍스트는 발음 기호와 완전히 일치하지 않는다.
- 프레임마다 정확한 정답을 주기 어렵다.
- 실제 음성에는 침묵, 잡음, 불필요한 구간도 함께 포함된다.

### 8. LibriSpeech 데이터셋
- **LibriSpeech** 는 약 1,000시간 규모의 깨끗한 영어 오디오북 데이터셋이다.
- 정제된 clean 데이터에서는 성능이 매우 높지만,
- 실제 noisy 환경이나 다양한 변수가 있는 상황에서는
  여전히 성능 개선 여지가 많다.

## 딥러닝 기반 음성인식

### 9. Acoustic Model
음성인식 모델은 음성 신호를 문자나 토큰으로 바꾸기 위해 음향 특징을 학습한다.

#### 모델 특징
- **Stride=2 1D Convolution**
  - 입력 길이를 줄이며 특징을 추출
- **Bidirectional LSTM**
  - 앞뒤 문맥을 함께 반영

#### 훈련 특징
- 입력 특징으로 **Mel-Spectrogram** 사용
- 대표적인 loss:
  - **CTC Loss**
  - **RNNT Loss**
  - **Cross Entropy Loss (with Attention)**

### 10. CTC
- RNN 기반 구조에서 많이 쓰이며,
  문자 사이에 **blank 토큰**을 두어 정렬 문제를 완화한다.

#### 장점
- 인코더 중심으로도 모델 구성이 가능하다.
- 어느 정도 정렬 위치를 추출할 수 있다.

#### 단점
- **Independence assumption** 한계가 있다.
- 성능 향상을 위해 별도의 **Language Model decoding** 이 필요한 경우가 많다.

### 11. LAS (Listen, Attend, Spell)
- Attention 기반 방식으로,
  텍스트를 생성할 때 입력의 어느 부분을 봐야 하는지 학습한다.
- 길이를 강제로 맞출 필요가 없다는 장점이 있다.

### 12. RNN-T
- 매 순간 출력할지, 유지할지를 함께 판단하는 구조이다.
- CTC의 blank와 유사한 개념이 있다.
- 보통 다음 3가지 흐름이 결합된다.
  1. 음성 step
  2. 텍스트 step
  3. 최종 예측 step

## Audio Auto Tagging

### 13. 오디오 자동 태깅 개념
- 어떤 소리는 특징이 뚜렷해서 비교적 쉽게 인식할 수 있다.
- 하지만 실제 환경의 다양한 소리를 모두 범용적으로 태깅하는 것은 어렵다.
- 많은 경우 직접 주석을 달거나 이벤트 단위로 라벨링하는 수작업이 필요하다.

### 14. 평가 지표
- **Label-weighted Label-Ranking Average Precision**
- 테스트 데이터에서 관련 레이블들의 순위를 얼마나 잘 매겼는지 평가하는 지표이다.
- 여러 실제 레이블의 순서를 고려하여 평균 정밀도를 계산한다.

## CTC 정렬 개념 이해

### 15. CTC alignment 핵심
- 모든 입력 프레임에 바로 문자를 할당하는 방식은 적절하지 않다.
- 예를 들어 **silence 구간**에도 문자가 강제로 붙는 문제가 생길 수 있다.
- 이를 해결하기 위해 허용된 출력에 새로운 토큰인 **epsilon(blank)** 을 도입한다.
- 입력 길이에 맞게 alignment 한 뒤,
  최종적으로 blank를 제거하여 실제 출력 시퀀스를 얻는다.
- 중요한 점은 **순서가 유지되어야 한다**는 점이다.
- 정렬은 many-to-one 형태로 생각할 수 있으며,
  하나 이상의 입력이 하나의 출력으로 매핑될 수 있다.

## 핵심 정리
- 음성은 시간축을 가진 신호이므로,
  단순 주파수 분석만으로는 부족하고 **시간 정보**도 중요하다.
- 그래서 음성 처리에서는 **STFT / Spectrogram / Mel-Spectrogram** 이 핵심이다.
- STT는 음성과 텍스트의 경계가 명확하지 않아
  일반적인 지도학습보다 더 어려운 문제다.
- 이를 해결하기 위해 **CTC, LAS, RNN-T** 같은 구조가 사용된다.
- 실제 음성 AI는 단순 텍스트 처리보다
  **신호 처리 + 딥러닝 + 정렬 문제**가 함께 들어가는 분야라는 점을 느꼈다.

## 느낀 점
- 음성인식은 단순히 말을 글자로 바꾸는 작업이 아니라,
  소리를 시간과 주파수 관점에서 해석한 뒤 텍스트와 정렬하는 복합적인 문제라는 점이 인상적이었다.
- 특히 CTC의 blank 개념이나 STFT처럼,
  실무에서 왜 이런 개념이 필요한지를 조금 더 이해할 수 있었다.
- 오늘은 개념이 많았지만, STT/TTS 실습을 같이 해서 흐름을 잡는 데 도움이 되었다.
