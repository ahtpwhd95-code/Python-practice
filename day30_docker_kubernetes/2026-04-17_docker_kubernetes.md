# 2026-04-17 Docker / Kubernetes / Network

## 오늘 학습한 내용

### 1. 다중 컨테이너 구조의 필요성
- API 서버(NodeJS)와 DB 서버(MySQL)를 하나의 컨테이너에 함께 넣을 수도 있지만,
  일반적으로는 **각 역할별로 컨테이너를 분리**하는 것이 더 적절하다.
- 하나의 컨테이너에서 여러 프로세스를 함께 실행하면 `ENTRYPOINT` 작성이 복잡해지고,
  각각의 실행 주기를 동시에 관리해야 해서 운영이 까다롭다.
- 반면 다중 컨테이너 구조는 **프로세스 실행 주기가 독립적**이어서 관리가 쉽다.

### 2. 다중 컨테이너 사용의 장점
- **독립된 관리**
  - API 서버와 DB 서버를 각각 별도로 관리할 수 있다.
  - 팀 단위 협업 시 역할 분리가 더 명확해진다.

- **효율적인 자원 할당**
  - 특정 서비스(API 등)에만 부하가 집중될 때 해당 컨테이너만 확장하기 좋다.

- **MSA와의 연결**
  - 기능별로 서비스를 분리하는 MSA 구조와도 잘 맞는다.

### 3. 쿠버네티스의 역할
- 쿠버네티스는 단순히 컨테이너를 실행하는 도구가 아니라,
  **여러 컨테이너를 관리하고 조정하는 오케스트레이션 플랫폼**이다.
- 다중 컨테이너 환경에서 배포, 확장, 복구, 관리를 자동화할 수 있다.

### 4. 도커 네트워크 개념
- 컨테이너는 기본적으로 서로 격리되어 있으므로,
  서로 통신하려면 **도커 네트워크**가 필요하다.
- 도커 네트워크를 사용하면 컨테이너 간 **IP 또는 DNS 기반 통신**이 가능하다.
- 대표적인 네트워크 드라이버:
  - `bridge`
  - `host`
  - `overlay`
  - `ipvlan`
  - `macvlan`
  - `none`

### 5. 기본 bridge 네트워크와 사용자 정의 bridge 네트워크 차이
- 별도 설정 없이 컨테이너를 실행하면 기본적으로 **기본 bridge 네트워크**에 연결된다.
- 기본 bridge 네트워크에서는 **IP 주소 기반 통신**은 가능하다.
- 하지만 **컨테이너 이름 기반 통신(DNS)** 은 지원되지 않는다.
- 사용자 정의 bridge 네트워크를 생성하고 같은 네트워크에 연결하면
  **컨테이너 이름으로 통신**할 수 있다.

### 6. 컨테이너 간 통신 실습 흐름
- nginx 컨테이너 두 개를 실행했을 때
  - 기본 bridge 네트워크에서는 IP로 통신 가능
  - 사용자 정의 bridge 네트워크에서는 컨테이너 이름으로도 통신 가능
- 실제 서비스 환경에서는 사용자 정의 네트워크가 더 직관적이다.

### 7. 파일 복사 / 이미지 저장 관련 명령어
- `docker cp`
  - 호스트와 컨테이너 사이 파일 복사

- `docker commit`
  - 실행 중인 컨테이너 상태를 새로운 이미지로 저장

- `docker save` / `docker load`
  - 이미지를 tar 파일 형태로 저장하고 다시 불러오기
  - 이미지 레이어와 메타데이터까지 유지

- `docker export` / `docker import`
  - 컨테이너 파일 시스템 스냅샷을 내보내고 가져오기
  - 주로 데이터 백업/이동 관점에서 볼 수 있음

## 핵심 정리

### 단일 컨테이너보다 다중 컨테이너가 유리한 이유
- 실행 주기 관리가 독립적이다.
- 역할 분리가 쉽다.
- 필요한 서비스만 확장 가능하다.
- MSA 구조와도 자연스럽게 연결된다.

### 기본 bridge vs 사용자 정의 bridge
- 기본 bridge:
  - IP 기반 통신 가능
  - 컨테이너 이름 통신 불가

- 사용자 정의 bridge:
  - IP 기반 통신 가능
  - 컨테이너 이름 기반 DNS 통신 가능

### save/load 와 export/import 차이
- `save/load`
  - 이미지 자체를 저장/복원
  - 레이어와 메타데이터 유지

- `export/import`
  - 컨테이너 파일 시스템만 스냅샷처럼 다룸
  - 이미지 히스토리 정보는 유지되지 않음

## 느낀 점
- 도커는 단순히 컨테이너 하나 실행하는 기술이 아니라,
  **여러 컨테이너를 어떻게 나누고 연결할지 설계하는 관점**이 중요하다고 느꼈다.
- 오늘 배운 다중 컨테이너 구조와 네트워크 개념은
  이후 쿠버네티스를 이해하는 데 기본이 될 것 같다.
